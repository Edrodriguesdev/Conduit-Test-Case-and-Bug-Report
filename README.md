Projeto Conduit - Estratégia de QA e Engenharia de Testes 🚀
Este repositório é uma vitrine técnica que demonstra minha transição de Desenvolvedor Front-end para Engenheiro de QA. Utilizei o projeto Conduit (RealWorld App) para aplicar metodologias de teste, análise de requisitos e validação de API.

Nota: Embora esta documentação principal esteja em português para facilitar a leitura do portfólio, todos os artefatos de teste, casos de uso e relatórios de bugs foram escritos em inglês, simulando um ambiente real de trabalho internacional.

📌 Visão Geral do Projeto
Como tecnólogo em Redes de Computadores e pós-graduando em Engenharia de QA, apliquei meu background técnico para garantir a qualidade da funcionalidade de Cadastro (Sign Up). O foco foi identificar bugs críticos e documentar cenários que garantam a integridade das regras de negócio.

📂 Estrutura do Repositório
De acordo com a organização do projeto:

User Story: Regras de negócio e especificações funcionais mapeadas.

Test Cases: Cenários detalhados (Caminho Feliz e Exceções) em formato técnico.

Bug Reports: Documentação de falhas críticas encontradas durante os testes.

API Testing: Validação de backend utilizando Postman com dados dinâmicos.

*   [**API Testing**](./documentation/api_testing.md): Validação de backend utilizando Postman. A suíte de testes cobre o ciclo completo de operações (CRUD) e autenticação, incluindo:
    *   **Registro e Login:** Validação de tokens e criação de usuários.
    *   **Gestão de Artigos:** Fluxos de criação (Create) e exclusão (Delete) de artigos.
    *   **Interação com Comentários:** Testes de criação e remoção de comentários em posts.

🛠️ Habilidades e Ferramentas Demonstradas
Mindset de QA: Testes exploratórios, testes negativos e análise de valor limite.

Documentação Técnica: Escrita de Bug Reports e Design de Casos de Teste (STLC).

Testes de API: Uso de Postman com variáveis de ambiente, payloads dinâmicos ({{$randomInt}}) e validação de Status Codes.

Background de Desenvolvedor: Análise de Logs no Console, inspeção de Network Tab e troubleshooting técnico de front-end.
